CLI для друку на AxiDraw

Огляд
 - Назва: cplot
 - Мова: C (GNU99)
 - Збірка: на основі Makefile (зовнішня система не потрібна)
 - Структура: `src/` вихідні коди, `lib/` об’єкти, `bin/` виконувані файли, `test/` тести cmocka

Швидкий старт
- Збірка: `make -j`
- Запуск: `./bin/cplot --help`
- Тести: `make tests`
- Valgrind: `make valgrind` (журнал у `log/valgrind.log`)
- Прибирання: `make clean`

Поширені сценарії
- Інтерактивний режим керування:
  - `./bin/cplot`
  - Далі використовуйте команди оболонки (`help`, `connect auto`, `pen up`, `jog 5 0` тощо).
- Сервер MCP для автоматизації:
  - `./bin/cplot --mcp`
  - Процес читає та пише повідомлення JSON-RPC 2.0 через stdin/stdout.

Редактор і інструменти
 - Конфігурація проєкту Zed у `.zed/`:
   - Завдання: build, run, tests, valgrind, clean
   - Налагодження: CodeLLDB для `bin/cplot`
   - Локальні налаштування: автоформатування C, відступ 4 пробіли, ліміт 100 символів
 - Конфігурація проєкту VS Code (за наявності) у `.vscode/`:
   - Завдання: `Build (make)`, `Run (bin/cplot)`, `Tests (make tests)`, `Valgrind (make valgrind)`, `Clean (make clean)`
   - Налагодження: конфігурації запуску для LLDB (macOS) і GDB (Linux), з попередньою збіркою через Build
   - Налаштування: форматування під час збереження, лінійки на 100 символів, відступ 4 пробіли; рекомендовані розширення `clangd` і CodeLLDB
 - Інструменти Clang:
   - `.clangd` примушує C‑режим і додає базові прапорці/шляхи для індексації/діагностики
   - `.clang-format` застосовує відступ 4 пробіли та ліміт у 100 символів (база GNU)
   - `.editorconfig` уніфікує відступи та переноси рядків

AI та Copilot у Zed
 - Вбудовані підказки (Edit Predictions): увімкніть GitHub Copilot глобально у вашому `settings.json` (відкрийте через `zed: open settings`):
   {
     "features": { "edit_prediction_provider": "copilot" }
   }
 - Аґентна розробка (Copilot Chat в Agent Panel): відкрийте панель (`cmd-?|ctrl-?`), перейдіть у налаштування панелі та увійдіть до GitHub Copilot Chat. Потім оберіть його в списку моделей.
   - Локальні проєктні налаштування не зберігають автентифікацію; вибір провайдера для Agent Panel краще робити через UI. Після входу Zed запам’ятовує ваш вибір.
 - Примітка: це користувацькі налаштування, у репозиторій не потрапляють.

Залежності
 - Компілятор: GCC або Clang з підтримкою C99/GNU99
 - Зневаджувач: LLDB (рекомендовано через CodeLLDB) або GDB
 - Необов’язково: Valgrind для перевірки витоків
- Необов’язково (тести): вимкнено для цієї збірки під macOS; зовнішніх залежностей немає

Застосовані практики якості
 - Окремі артефакти під `bin/`, `lib/` і `log/`
 - Суворі попередження та захист стека увімкнені за замовчуванням у `Makefile`
 - Конфігурація редактора/форматера зафіксована для відтворюваного форматування
 - Clangd налаштовано для надійної індексації без `compile_commands.json`
 - Завдання та налагодження Zed — для швидкого циклу розробки й зручного дебагу

Примітки
- `project.conf` містить `BINARY := cplot`. Змініть за потреби, якщо перейменуєте бінарник.
- Якщо згодом перейдете на CMake, увімкніть `CMAKE_EXPORT_COMPILE_COMMANDS=ON` і приберіть зайві прапорці з `.clangd`.
- Тести вимкнено; зовнішні залежності відсутні.
- Вибір пристрою відбувається в інтерактивній оболонці командою `model <name>` (наприклад, `model minikit2`).
- Керування рухом, що конфігурується: планувальні подача/прискорення читаються з конфігурації та обмежуються можливостями пристрою.
